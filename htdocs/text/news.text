13 April 2013
Avoid Using GParted 0.15.0 for Partition Moves, Resizes, Copies and Checks
<br>
We are investigating reports of intermittent crashes with GParted
0.15.0.<br>
See
<a href="https://bugzilla.gnome.org/show_bug.cgi?id=697727">Bug
697727 - Segfault in livecd Gparted v 0.15.0-3 when copying
partition</a>.<br>
<br>
The risk of <b><u>LOSS of DATA</u></b> is highest when performing
partition moves, resizes, copies, and checks.  As such we recommend
using GParted 0.14.1 for all move, resize, copy, and check
operations.<br>
<br>
Other operations such as create, delete, format, label, set UUID, and
delete might also crash, but these should not result in unexpected
data loss, and the operation can be repeated.<br>
<br>
Until this problem is resolved we advise using the last stable release
of GParted Live, which
is <a href="http://sourceforge.net/projects/gparted/files/gparted-live-stable/0.14.1-6/">GParted
Live 0.14.1-6</a>.<br>
<br>
We highly recommend that you <b><u>BACKUP your DATA</u></b> before
using <b><i>any partitioning tool</i></b>.<br>
<br>

partition size.  Thanks to work by Mike Fleetwood this resolves <a

can then be put to good use.<br>

href="https://bugzilla.gnome.org/show_bug.cgi?id=678831">bug
#678831</a> which resolves a problem with overlapping partitions when
resizing an extended partition.<br>
<br>

<ul>
<li>Adds web browser icon to desktop</li>

(<a href="https://bugzilla.gnome.org/show_bug.cgi?id=578842">bug
#578842</a>)<br>

<li>Updates Linux kernel to 3.2.21-3</li>
<li>Based on Debian Sid repository (as of 2012/Jul/14)</li>
<li>Includes new GParted 0.13.0 release</li>

release.<br>

18 July 2011
GParted 0.9.0
<br>
The most significant change in this release is the ability to
compile and link with libparted 3.0.<br>
<br>
GParted retains full functionality when compiled and linked with
libparted versions prior to 3.0, for example libparted-2.4.<br>
<br>
NOTE:  Because Parted 3.0 removed some file system specific functionality,
the following abilities are lost or changed when GParted is compiled
and linked with libparted 3.0:<br>
<ul>
<li>Unable to grow and shrink FAT16 and FAT32 file systems</li>
<li>Unable to shrink HFS and HFS+ file systems</li>
<li>Unable to determine used and unused sectors in HFS and HFS+ file systems
</li>
<li>Unable to erase file system signatures on partition create and format
</li>
<li>FAT16 and FAT32 file system moves performed internally by
    GParted instead of by libparted</li>
</ul>
Other key changes include:
<ul>
<li>Fix size off by one sector with unrecognized disk label</li>
<li>Fix GParted crashes with assertion (head_size <= 63)
  <ul>
  <li>Requires libparted-2.4 or higher</li>
  </ul>
</li>
</ul>
See the <a href='https://sourceforge.net/projects/gparted/files/gparted/gparted-0.9.0/gparted-0.9.0-README.md/view'>Release Notes</a> for more details.
<br><br>

GParted 0.6.4
<br>
This release of GParted fixes a regression introduced with the
previous version.  The problem introduced was that when

2 May 2010
Suggested Patches for Parted 2.2
<br>
It appears that the Parted 1.9.0 through 2.2 releases have lost some
capability that earlier releases (1.8.8 and earlier) possessed.<br>
<br>
These newer Parted releases are unable to inform the kernel of
partition changes if another partition is mounted, or if an active LVM
partition is on the same device.<br>
This latter case was reported in the following bug report:<br>
<a href="https://bugzilla.gnome.org/show_bug.cgi?id=608712">Bug 608712 - GParted fails to edit/delete partitions if active LVM on same disk device</a><br>
<br>
Fortunately, the parted git repository contains updates that restore
this capability.<br>
<br>
In light of this and potential problems with dmraid, Steven Shiau has
kindly updated GParted Live 0.5.2-9 with the following three
patches:<br>
<ul>
<li><a href="http://git.debian.org/?p=parted/parted.git;a=commit;h=c1eb485b9fd8919e18f192d678bc52b0488e6ee0">libparted: don't canonicalize /dev/mapper paths
</a></li>
<li><a href="http://git.debian.org/?p=parted/parted.git;a=commit;h=0e04d17386274fc218a9e6f9ae17d75510e632a3">libparted: reenable use of BLKPG ioctls</a></li>
<li><a href="http://git.debian.org/?p=parted/parted.git;a=commit;h=7165951dfb584aae2901ac3f1a28fe3624667f19
">libparted: improve BLKPG error checking</a></li>
</ul>
The above three patches improve support for DMRAID, and permit editing
unmounted partitions on devices that have at least one mounted
partition and/or an active LVM partition.<br>
<br>
We recommend the above three patches be applied to parted-2.2 if you
are compiling your own copy of Parted or using libparted with GParted.

26 March 2008
GParted 0.3.6
<br>
GParted now has support for partition labels (also known as volume labels).<br>
To change partition labels, use the GParted menu option "Partition -> Label".
<br><br>
Full partition label support (read and write) is available for the following file systems:
<br> &nbsp  ext2, ext3, fat16, fat32, jfs, ntfs, reiserfs, and xfs.
<br> Partial support (set volume label on filesystem creation only) is available for:
<br> &nbsp hfs, linux-swap, and reiser4.
<br><br>
New Dependencies (Optional):
<ul>
<li><a href="http://mtools.linux.lu/">mtools</a> required to read and write fat16 and fat32 labels.</li>
<li>vol_id command required to read linux-swap labels.</li>
</ul>
See the <a href='https://sourceforge.net/project/shownotes.php?group_id=115843&amp;release_id=587229'>Release Notes</a> for more details.
<br><br>
Curtis

03 December 2004
GParted 0.0.7
<p>
gparted-0.0.7 is finally here. It took some time, because i had to rewrite a lot of code.
Every filesystem now has its own class and is responsible for its own creation, resizing etc..
This makes it very easy to add more filesystems to gparted and that's what we all want :)<br>
<br>
Because the ext2/3 code of libparted was slightly outdated and sometimes caused trouble, gparted now
uses the e2fstools to handle them. Same goes for reiserfs, you now need the official tools from namesys to handle reiserfs partitions.<br>
<br>
Since most (all) of these tools lack decent API's i just call the commands from the code, this is a bit scary and definitely not considered
elegant coding, but i didn't have much choice and so far it works out quite well.<br>
<br>
At startuptime gparted scans for available commands and enables/disabled certain functionality. e.g. if 'mke2fs' is found, creation of ext2 is enabled.
This is quite cool since you can now enable support for filesystems without rebuilding gparted. just install the tools for your specific filesystem en (re)start gparted. That's all :)<br>
<br>
Of course using CLI-tools has some disadvantages. One of the most annoying is the lack of proper progressfeedback. Although i have some ideas on how to solve this, atm it's still just a dumb guess :)<br>
<br>
More information can be found in the  <a href="http://sourceforge.net/project/shownotes.php?release_id=287614">Changelog</a><br>
<br>
Have fun and don't forget to report your bugs :^)
</p>

24 October 2004
GParted 0.0.6
<p>
A brand-new GParted =)<br>
We now have support for <b>reiserfs</b> [1]<br>
Startup time is down with a couple of seconds (the gui gets loaded first and the deepscan of all partitions is performed later)<br>
Changes, Fixes, new and updated translations.... <br>
<br>
Since nothing else comes to mind i happily point at the <a href="http://sourceforge.net/project/shownotes.php?release_id=277487">Changelog</a> :P
<br>
<br>
<br>
[1] Besides it's builtin filesystems, libparted also offers support for reiserfs through an extern package (progreiserfs).
The presence of this package is detected at runtime and support for reiserfs is enabled/disabled accordingly.<br>
Since progreiserfs is NOT the 'official' reiserfsprogs it's a good idea to use caution!<br>
Get it from <a href="ftp://ftp.namesys.com/pub/libreiserfs/progsreiserfs-0.3.0.5.tar.gz">
ftp://ftp.namesys.com/pub/libreiserfs/progsreiserfs-0.3.0.5.tar.gz
</a>
</p>
