Introduction

GParted Live is a small bootable GNU/Linux distribution for x86 based computers. GParted Live enables you to use all the features of the latest versions of the GParted application.

This manual describes how to use the GParted Live image, from boot up to shut down.

Tip

To learn how to write the GParted Live image to removable media, such as CD-ROM or a USB flash drive, see http://gparted.org/livecd.php.

Booting GParted Live

This section describes how to boot up GParted Live on your computer.

If your computer does not boot from CD-ROM discs or USB flash drives, then see the section called “Appendix A: Preparing BIOS”.

Booting from CD-ROM or USB Flash Drive

To start GParted Live:

  1. Power on your computer with the media containing GParted Live.

    • If using a USB flash drive, insert the USB flash drive into an open USB port and turn on the power for your computer.

    • If using a CD-ROM disc, turn on the power for your computer and quickly insert the CD-ROM disc in the CD-ROM tray of your computer.

  2. To use the default settings, press the Enter key when the GParted Live boot screen is displayed.

    Figure 1. GParted Live Boot Screen


    Tip

    To use a different mode of GParted Live, use the Up arrow key or Down arrow key to select the mode. Press the Enter key to activate the mode.

    For example, to use your CD-ROM R/W tray after booting GParted Live select the Other modes of GParted Live entry, and then select the To RAM entry.

    To return to the default GParted Live boot screen, press the Escape key.

    Figure 2. Other modes of GParted Live


  3. To use the standard US keymap, press the Enter key.

    If you require a different keymap then use the Up arrow, Down arrow, and Enter keys to choose an appropriate keymap for your keyboard.

    Figure 3. Keymap Question


  4. To use the US English language, press the Enter key.

    If you desire a different language then type the number shown beside the langauge and press the Enter key.

    Figure 4. Language Question


  5. To start up the default graphical environment and begin using the GParted application, press the Enter key.

    Figure 5. Video Mode Question


  6. You can now begin using the GParted application.

    Figure 6. Graphical Desktop with GParted


    Note

    If your computer is unable to display the default graphical environment, then see the section called “Appendix B: Tips on Booting GParted Live”.

Using GParted Live

This section describes how to use some of the features of GParted Live.

Using GParted Application

For instruction on how to use the GParted application, please refer to the GParted Application Help Manual.

The GParted Manual is available on the documentation page of the GParted web site.

Working with a Terminal Window

Many commands in GNU/Linux are executed in a terminal window.

Following is a brief description of how to open a terminal window, enter commands, and close the terminal window.

  1. To open a terminal window, double-click the Terminal icon in the upper left hand side of the desktop.

    Figure 7. Terminal Window


  2. You can enter commands in the terminal window. Commands are entered immediately following the user@debian$ prompt.

    For example, to list the files in a directory with the "ls" command, enter: 

    user@debian:~$ ls

  3. To close the terminal window, enter: 

    user@debian:~$ exit

Capturing a Screenshot

To capture a graphic image of a window in a file:

  1. Double-click the Screenshot icon in the upper left hand side of the desktop.

  2. Click the Yes button. When the mouse pointer changes to a cross-hair, click on the window you wish to capture.

    Figure 8. Screenshot Prompt


  3. A screenshot preview window is displayed. A screenshot acknowledgement window is also displayed that indicates the screenshot is saved in the file /home/user/gparted.jpeg.

    Figure 9. Screenshot Saved


    Note

    Files saved to the root file system in a GParted Live session are lost when you shut down the computer.

    To copy files to more permanent storage see the section called “Copying Files to Other Media”.

  4. To close the screenshot acknowledgement window click the OK button.

  5. To close the screenshot preview window click the X close window icon in the upper right hand corner of the window.

Saving GParted Details

Tip

Providing the gparted details log file is very helpful when reporting bugs, or when asking for help in the forum.

To save the details from applying operations:

  1. Click the Save Details button in the Applying pending operations window.

    Figure 10. Applying Pending Operations Window


  2. Click the Save button to save the operation details in the file /root/gparted_details.htm .

    Figure 11. Save Details Window


    Note

    Files saved to the root file system in a GParted Live session are lost when you shut down the computer.

    To copy files to more permanent storage see the section called “Copying Files to Other Media”.

  3. To close the applying pending operations window click the Close button.

Copying Files to Other Media

In a GParted Live session the root file system exists in RAM memory. All files saved to this file system will be lost when you shut down the computer.

Following are instructions on how to copy files to more permanent storage media.

Note

A terminal window is required for the following instructions. See the section called “Working with a Terminal Window”.

Copying to USB Flash Drive

  1. Create a mount point directory by entering: 

    user@debian:~$ sudo mkdir /tmp/myusb

  2. Insert the USB flash drive into an open USB port.

  3. To determine the USB device name, enter: 

    user@debian:~$ dmesg

    Figure 12. Command dmesg Output


  4. Visually search through the last few lines of dmesg output to find the device name. The device name should be in the form "sdX" where "X" is a lower case letter.

    In the above example, the device name is "sdc".

  5. Mount the USB flash device.

    For example, if the device name is "sdc", enter: 

    user@debian:~$ sudo mount /dev/sdc /tmp/myusb

  6. To copy a file, such as /root/gparted_details.htm, enter: 

    user@debian:~$ sudo cp /root/gparted_details.htm /tmp/myusb

  7. When you are finished copying files, unmount the USB flash drive by entering: 

    user@debian:~$ sudo umount /tmp/myusb

  8. Remove the USB flash drive from the computer.

Copying to Hard Drive Partition

  1. Create a mount point directory by entering: 

    user@debian:~$ sudo mkdir /tmp/myhd

  2. Mount the hard drive partition. For example, to mount the first partition sda1:

    • To mount an NTFS partition, enter: 

      user@debian:~$ sudo mount -t ntfs-3g /dev/sda1 /tmp/myhd

    • To mount other partitions, such as ext2 or fat32, enter: 

      user@debian:~$ sudo mount /dev/sda1 /tmp/myhd

  3. To copy a file, such as /root/gparted_details.htm, enter: 

    user@debian:~$ sudo cp /root/gparted_details.htm /tmp/myhd

  4. When you are finished copying files, unmount the hard drive partition by entering: 

    user@debian:~$ sudo umount /tmp/myhd

Copying to Floppy Disk

  1. Create a mount point directory by entering: 

    user@debian:~$ sudo mkdir /tmp/myfloppy

  2. Insert the floppy disk into the floppy drive.

  3. Mount the floppy disk by entering: 

    user@debian:~$ sudo mount /dev/fd0 /tmp/myfloppy

  4. To copy a file, such as /root/gparted_details.htm, enter: 

    user@debian:~$ sudo cp /root/gparted_details.htm /tmp/myfloppy

  5. When you are finished copying files, unmount the floppy disk by entering: 

    user@debian:~$ sudo umount /tmp/myfloppy

  6. Eject the floppy disk from the floppy drive.

Changing Screen Resolution

Caution

Do not change the screen resolution while GParted is applying operations. This might cause loss of data.

To change the screen resolution:

  1. Double-click the Screen resolution icon at the top of the desktop.

  2. Choose a screen resolution and refresh frequency by clicking the Modes drop down list button and selecting an appropriate entry.

    Figure 13. Screen Resolution Window


  3. Click the Apply button.

Enabling Network Access

To enable wired ethernet TCP/IP networking:

  1. At the terminal prompt enter: 

    user@debian:~$ sudo dhclient eth0

  2. Wired ethernet TCP/IP networking should now be enabled.

    Network commands, such as "scp" to securely copy files, should now work properly.

Finishing GParted Live Session

This section describes how to finish your GParted Live session.

Shutting Down GParted Live

To shut down GParted Live:

  1. Double-click the Exit icon in the upper left hand side of the desktop.

  2. Select the Shutdown entry and click OK.

    Figure 14. Shutdown Prompt


  3. When prompted, remove the CD-ROM disc or USB flash drive and press the Enter key.

    Figure 15. Remove Disc Prompt


  4. Your computer will shut down.

Appendix A: Preparing BIOS

To boot the GParted Live image from USB flash drive or CD-ROM disc, your computer BIOS must be properly configured. More specifically the BIOS must be set to boot from CD-ROM and other removable devices before booting from hard disk drives.

Entering BIOS Setup

To enter the BIOS setup screen on you computer:

  1. Turn on the power for your computer.

  2. Watch for a screen indicating which key to press to enter the BIOS setup screen.

    For example at the bottom of the following screen shot, the F2 is used to enter the BIOS setup screen. On other computers it might be the Delete key, the Insert key, the F9 key, or some other key.

    Figure 16. BIOS Setup Prompt


    Note

    If you do not see a screen that indicates the key to press to enter BIOS setup, then you might need to consult the documentation that came with your computer.

  3. Press and hold the appropriate key, noted from above, to enter BIOS setup.

Setting Boot Order in BIOS

  1. Read the BIOS setup screen instructions to learn how to navigate the menu options and how to change settings.

  2. Change the BIOS boot order so that CD-ROM and removable devices, such as a USB flash drive, are earlier in the boot order than hard disk drives.

    Figure 17. BIOS Setup Boot Order


Saving BIOS settings

  1. Follow the on-screen instructions to save the BIOS settings.

    Figure 18. BIOS Save Settings


  2. Your computer will reboot after saving the BIOS settings.

Appendix B: Tips on Booting GParted Live

Are you experiencing trouble trying to boot the GParted Live image?

If so then here are some tips that might help.

List of Booting Tips

  1. Try using one of the settings under "Other modes of GParted Live", such as:

    • GParted Live (Safe graphic settings, vga=normal)

    • GParted Live(Failsafe mode)

  2. Try using 'Forcevideo' to configure X manually.

    While booting up, select option "(1) Run 'Forcevideo' to config X manually".

    • Try a lower screen resolution.

    • Try a driver appropriate for your video card. Vesa should work with many video cards.

    • Try a low color depth if your video card does not have much RAM memory.

  3. While one of the settings is highlighted, press the Tab key to display the list of boot options.

    1. If you have Intel i915 graphics, try adding one of:

      • i915.modeset=0

      • i915.modeset=1

    2. If you have ATI Radeon graphics, try adding one of:

      • radeon.modeset=0

      • radeon.modeset=1

    3. If you have NVidea graphics, try adding one of:

      • nouveau.modeset=0

      • nouveau.modeset=1

    4. Try removing all of the modeset parameters listed above.

  4. If you have an IBM NetFinity 5000, GParted might stop booting with "Unable to find a medium containing a live file system"

    There's a known issue on this hardware from IBM with the IDE CD-ROM. Add "all_generic_ide" as kernel parameter, i.e. press Tab at the beginning of CD-ROM booting splash screen.

    On an IBM NetFinity 5000 server, the GUI starts with a low video resolution. The S3 based onboard video on this IBM server hardware offers 1 MB video RAM only. Use the following extra step to select a higher video resolution:

    • Use option "1" (configuring X manually) and set the parameters to 1024x768 and 256 colors (8 bpp).

If you are still experiencing difficulty booting GParted Live, then please create a new post in the "Live Media" section of the GParted forum and describe your situation.
